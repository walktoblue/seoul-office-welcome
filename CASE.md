## 한 줄 요약
한동대 서울사무소 담당자가, 방학마다 새로 오는 근로 아르바이트 학생에게 매번 구두로 반복하던 온보딩 교육을, 학생이 스스로 진행하는 체크리스트 웹앱으로 바꿨다.

## 자동화 전 — 원래 어떻게 했나
- **하던 일**: 방학이 시작되고 새 근로학생이 첫 출근하면, 서울사무소 소개·사무실 안내·후원업무 오리엔테이션·각종 시스템 등록(한국장학재단·근로시스템)·안전교육 이수 확인·시간표 제출·프린터 설치 안내 등을 담당자가 하나하나 말로 설명했다. 근무 중에도 매월 시간표 제출, 갈대상자 책 독후감, 프로파일링 교육, 라벨지 인쇄 실습을 그때그때 다시 안내했다.
- **불편했던 점**: 방학마다 학생이 바뀌어 같은 설명을 처음부터 반복해야 했고, 담당자가 자리를 비우면 교육이 밀렸다. 학생이 어디까지 안내받았는지 기억에 의존해 확인해야 했다.

## 무엇을 자동화했나
- **앱이 대신하는 일**: 학생이 연도·학기·학번·이름만 입력하면 hello(초기 안내)·main(근무 중 업무)·good bye(마무리) 순서의 체크리스트가 뜨고, 실제로 업무를 진행할 때마다 스스로 체크한다. 업무마다 "설명" 버튼으로 상세 안내를 언제든 다시 볼 수 있어 담당자가 옆에서 말로 반복할 필요가 없다. 같은 정보로 다시 접속하면 이전 진행 상황을 그대로 이어간다.
- **일부러 사람한테 남긴 일**: 실제로 업무를 했는지 확인하는 체크 자체는 학생 스스로 하게 뒀다(자기 신고). 담당자는 최종적으로 관리자 대시보드에서 학생별 진행 현황을 확인하고, 누락된 부분이 보이면 직접 챙긴다.

## 사람이 개입하는 지점 (Human-in-the-Loop)
- 학생이 정말 업무를 했는지는 시스템이 검증하지 않는다 — 자기 신고 체크리스트이기 때문에, 담당자가 관리자 대시보드의 "자세히 보기"로 학생별 완료·미완료 항목을 직접 확인해 누락을 챙긴다.
- 업무 내용이 바뀌면(새 제도 도입, 절차 변경 등) 담당자가 관리자 대시보드에서 업무 설명을 직접 수정하거나 새 업무를 추가한다 — 자동으로 바뀌지 않는다.

## 업무 흐름 — 전 → 후
- **전**: 학생이 첫 출근할 때마다 담당자가 옆에서 말로 하나씩 설명하고, 진행 상황을 기억이나 메모로 관리했다.
- **후**: 학생이 웹앱에서 스스로 체크리스트를 따라가고, 담당자는 관리자 대시보드에서 진행 현황을 한눈에 확인만 하면 된다.

## 다시 만든다면
- 학생 식별을 (연도·학기·학번·이름) 조합으로 했는데, 처음부터 학번을 넣지 않았다가 나중에 추가했다 — 비슷한 반복형 등록 앱을 만든다면 "같은 사람을 어떻게 구분할지"부터 먼저 정하고 시작하는 게 낫다.
- 체크리스트 항목이 아직 몇 십 개 수준이라 괜찮지만, 항목이 훨씬 많아지면 세션별 진행률뿐 아니라 "이번 주에 해야 할 일" 같은 마감 기반 뷰가 있으면 더 좋을 것 같다.
